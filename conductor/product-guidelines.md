# Product Guidelines

## Core Principles
Terminal TreeView Navigator is designed as an essential daily utility, prioritizing speed, reliability, and precision. It should feel like a natural extension of the terminal, providing a clear and efficient way to navigate the filesystem.

### 1. Keyboard-First Efficiency
- All primary actions (navigate, select, expand, collapse) must be accessible through single-key shortcuts.
- Minimize the number of keystrokes required for any given operation.
- Avoid forcing users to leave the home row unless necessary.

### 2. Strict Minimalist UI
- Maintain a clean and uncluttered interface.
- Only display information that is immediately relevant to the user's current task.
- Use visual elements (icons/colors) purposefully to enhance readability, not just for decoration.

### 3. Transparent & Reactive UI
- Provide immediate visual feedback for every user input.
- Clearly indicate the current state of the application (e.g., loading, error, success).
- Ensure that the interface reflects the state of the filesystem in real-time or near real-time.

## Prose & Communication Style
The tool communicates in a minimalist and technical tone, focusing on clarity and functionality over conversational fluff.

- **Conciseness:** Use short, punchy phrases for all UI messages and logs.
- **Directness:** State errors and status updates clearly and without apology.
- **Technical Accuracy:** Use precise terminal and filesystem terminology.

## Branding & Personality
- **Tone:** Reliable, no-nonsense, and efficient.
- **Identity:** An essential daily utility that 'just works' and stays out of the way until needed.
